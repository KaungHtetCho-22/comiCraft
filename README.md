## What is comiCraft?

**comiCraft** is a browser-based creative tool that transforms text prompts into fully-rendered, multi-page comic storyboards вҖ” powered by Google Gemini's multimodal AI.

Designed for writers, artists, and storytellers who want to **visualize ideas fast** without sacrificing consistency across pages.

> **The core problem with AI comic generation:** Characters look completely different by page 3.
> **comiCraft's answer:** My Studio вҖ” a consistency engine that locks character and environment references into every generation call, keeping your visual universe coherent from panel one to the end.

<br/>

| рҹҺЁ | **Page Rendering** | Render storyboard sketches into finished pages across multiple art styles |

## рҹҡҖ Quick Start

### Prerequisites

| Requirement | Details |
|:---|:---|
| **Python 3.12+** | Runtime for the backend |
| **[uv](https://github.com/astral-sh/uv)** | Fast Python package manager |
| **Google Gemini API Key** | Powers all text and image generation |
| OpenAI-compatible endpoint *(optional)* | Alternative text generation path |

<br/>

### вҡЎ One-Command Start

```bash
chmod +x start.sh && ./start.sh
```

This script will automatically:

1. Install all backend dependencies via `uv`
2. Start the **Flask backend** вҶ’ `http://localhost:5003`
3. Start the **frontend server** вҶ’ `http://localhost:8000`

Then open **[http://localhost:8000](http://localhost:8000)** in your browser.

## рҹ•№пёҸ How to Use

### Creating a New Story

```
 в‘  Write a prompt   вҶ’   в‘Ў Set Pages & Style   вҶ’   в‘ў Click Run   вҶ’   в‘Ј Render Pages
```

1. Write your story idea in the main text area
2. Set the number of **Pages** and **Rows per Page**
3. Choose a **Comic Style**
4. Click **Run** вҖ” storyboard panels are generated instantly
5. Click **Render Page** on any page, or **Render All** to produce all final images

<br/>

### Continuing an Existing Story

```
 в‘  Adjust the Pages slider   вҶ’   в‘Ў Click "Continue +N"   вҶ’   в‘ў New pages appended
```

comiCraft synthesizes existing narrative context and extends the story seamlessly вҖ” visuals and plot stay coherent.

## рҹ“қ Notes

- Generated images are stored at `backend/static/images/` вҖ” clear periodically to free disk space
- All session data and Studio items live in **browser local storage** вҖ” no backend persistence required
- Missing `refer_image/<style>/` directories are non-fatal; override the path with `COMICCRAFT_REFER_IMAGE_BASE_PATH`
- Chrome may request `/.well-known/appspecific/com.chrome.devtools.json` вҖ” the resulting 404 is harmless
